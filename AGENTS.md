# 协作说明

本仓库用于维护中文考研数学 Obsidian 复习笔记和错题记录。修改时请保持仓库精简，不再添加非笔记类功能。

## 内容范围

- `高等数学/`、`线性代数/`、`概率论/`：学科专题笔记。
- `方法库/`：跨专题复用的解题方法。
- `错题/`：错题模板、错题复盘和阶段性回顾。
- `概率论/概念/`：概率论中真正需要反复跳转复习的概念卡片。

不要恢复或新增以下非核心功能：

- 自动练习卷、生成脚本、测试脚本、结构化 JSON 题库。
- 与复习笔记无关的工具目录或缓存目录。

## 笔记整理原则

- 所有面向学习者的内容使用中文。
- 笔记优先服务 Obsidian 阅读和复习，不写成代码项目说明。
- 使用中文文件名和中文标题。
- 新章节优先建立在对应学科目录下，例如 `概率论/多维随机变量及其分布.md`。
- 章节笔记要适合直接复习，结构建议包括：
  - 开头说明本章定位和核心任务。
  - “基础知识结构”总览。
  - 按知识点分节整理定义、公式、性质、题型。
  - “常见题型”。
  - “常见失分点”。
  - “复习建议”。
  - “关联地图”。
- 如果用户提供教材图片，优先按图片中的章节框架整理，再补充必要解释、公式、题型和易错点。
- 新增错题时优先复制 错题模板 的结构，记录错误原因和复习日期。

## 双向链接规则

- 不要把所有关键词都机械加成双向链接；过多行内链接会变成“加粗提醒”，降低阅读体验。
- 正文中只链接真正有复习价值、会反复跳转、且已经有内容或准备建立内容的核心概念。
- 对于重要概念，优先建立“概念卡片”，而不是只留下空链接。
- 概念卡片建议放在对应学科的 `概念/` 子目录下，例如 `概率论/概念/条件概率.md`。
- 概念卡片建议包含：
  - 定义
  - 核心公式
  - 适用信号
  - 常见错误
  - 相关链接
- 章节底部要用“关联地图”表达知识关系，而不是只在正文里堆链接。
- 关联地图可以按主线组织，例如：
  - 概率公式主线：条件概率 → 全概率公式 → 贝叶斯公式。
  - 离散型主线：分布律 → 二项分布、泊松分布。
  - 连续型主线：概率密度 → 正态分布。
- 添加或移动链接后，应检查明显断链；若链接目标只是临时提醒，宁可先不加。

## 概率论笔记风格

- 概率论章节按教材顺序建立主干：
  1. 随机事件与概率
  2. 一维随机变量及其分布
  3. 多维随机变量及其分布
  4. 后续章节继续补充。
- 每章先整理“本章解决什么问题”，再展开概念和公式。
- 公式必须配合适用条件和题型信号，避免只罗列公式。
- 对易混点要明确区分，例如：
  - 互斥与独立。
  - 联合分布与边缘分布。
  - 分布函数与概率密度。
  - 二项分布与超几何分布。
- 概率论常见错误要引导记录到 错题模板，并链接回相关概念卡片。

## 数学公式规范

- 行内公式只用于短表达式，例如 `$f(x)$`、`$a \ne -1$`。
- 长公式使用独立行间公式：

```markdown
$$
\int_a^b f(x)\mathrm{d}x
$$
```

- 行间公式的开始和结束分隔符 `$$` 必须分别单独占一行。
- 积分微分统一使用 `\mathrm{d}x`、`\mathrm{d}t`、`\mathrm{d}u`。
- 多行等式优先使用 `aligned` 环境，避免把 `=`、`+`、`-` 单独放一行。
- 分段函数使用 `cases` 环境，保持 Obsidian 和 GitHub 都易读。

- 不提交临时文件、缓存文件或与复习无关的产物。
- 保持提交内容聚焦，避免无关重构。
